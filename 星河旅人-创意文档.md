# 星河旅人 — 创意文档

> AI 驱动的自适应太空逃亡游戏 · TRAE AI 创造力大赛参赛作品

---

## 一、创意名称与介绍

### 创意名称
**星河旅人** — AI 自适应难度太空逃亡游戏

### 想解决什么问题

传统休闲躲避类游戏的难度曲线是固定的，高手玩几局就觉得无聊，新手又容易因为太难而放弃。玩家很难在游戏中持续获得"刚刚好"的挑战感，导致玩几轮就腻了。

同时，市面上同类游戏玩法高度同质化，缺少真正"懂玩家"的游戏体验——每一局都是相似的陨石下落模式，玩多了就有套路，新鲜感快速流失。

### 为什么会想到做这个

最初做"太空逃亡"是想做一个简单解压的小游戏，但身边朋友反馈差异很大：有的说太简单玩两局就腻了，有的说太难根本活不过 10 秒。于是想到，如果游戏能像个会观察的对手，根据玩家的水平实时调整难度，让每个人都能玩得爽、还想再来一局，会不会很有意思？

加上"AI 创造力大赛"的契机，就想把这个想法打磨成一个完整的作品——用 AI 的思路做一款真正"懂玩家"的小游戏。

### 大概是什么产品

一款基于 Canvas 的网页休闲游戏，玩家操控飞船躲避陨石，游戏内置 AI 难度系统，会根据玩家的躲避率、移动轨迹实时调整陨石密度和速度，让每一局都有刚刚好的挑战感，配合全服排行榜和皮肤收集，打造让人"上瘾"的游戏体验。

## 二、目标用户及痛点

### 面向哪些用户

- **核心用户**：18-35 岁喜欢玩休闲小游戏的年轻人
- **泛用户**：工作学习间隙想放松几分钟的上班族、学生党
- **潜在用户**：对太空题材、像素风有偏好的游戏爱好者

### 在什么场景下使用

- 通勤路上、排队等候时，随手玩个 2-3 分钟
- 工作/学习累了，打一局换换脑子
- 睡前躺床上，挑战一下今日最高分
- 和朋友闲聊时，比一比谁的分数高

### 当前痛点

| 痛点 | 具体表现 |
|------|---------|
| **难度不对味** | 简单的玩两局就腻，难的又打不过，很难找到刚好的挑战感 |
| **重复感强** | 每局陨石模式都差不多，玩几次就摸清规律了 |
| **没有攀比感** | 单机游戏，分数只是数字，没人比，就没动力冲高分 |
| **成长感弱** | 除了分数没有别的追求，解锁内容少 |

## 三、价值与意义

### 体验价值：让每个玩家都能获得"心流"体验

通过 AI 自适应难度系统，游戏能实时感知玩家的水平，动态调整难度——新手不会秒退，高手也不会无聊。玩家总能在"差一点点就破纪录"的状态里，获得沉浸式的心流体验。这是固定难度游戏做不到的。

### 创作价值：探索小游戏 + AI 的新可能

现在很多人一提到 AI 游戏就是"AI 生成剧情""AI NPC"，但很少有人关注"AI 难度设计"这个方向。这个项目想探索一下：用 AI 的思路做难度，能不能让一个简单的躲避类游戏变得更耐玩、更懂玩家？项目代码开源，也能给其他独立游戏开发者提供参考。

## 五、核心玩法说明

### 基础操作

- **桌面端**：方向键 ↑↓←→ 控制飞船上下左右移动
- **移动端**：手指在屏幕上滑动，飞船跟随手指移动

### 游戏流程

```
开始游戏 → 飞船位于屏幕底部中央 → 陨石从顶部不断下落 →
玩家操控飞船躲避 → 每躲过一颗陨石 +10 分 →
分数越高，难度越大 → 碰到陨石游戏结束 → 显示得分和排名 →
再来一局 / 查看排行榜
```

### 让玩家"上瘾"的关键设计

**1. "差一点点"的心流**
AI 难度会把玩家控制在"有点挑战但能撑住"的状态里，每次死掉都觉得"刚才再快一点就好了"，于是忍不住再来一局。

**2. 全服排行榜的攀比欲**
看到排行榜上别人的分数，就想超过去。每次刷新自己的最高排名，都有强烈的成就感。

**3. 皮肤收集的乐趣**
不同分数段解锁不同的飞船皮肤，有目标感，也有收集欲。

**4. 单局时长短**
一局通常 1-3 分钟，随时能玩、随时能停，没有负担。

## 六、AI 自适应难度设计

### 核心思路

不是简单的"分数越高 → 难度越高"，而是**实时分析玩家的操作数据，动态调整难度**，让玩家始终处于"跳一跳够得着"的最佳挑战状态。

### 监测的数据指标

| 指标 | 说明 | 反映什么 |
|------|------|---------|
| **躲避率** | 一段时间内成功躲避的陨石数 / 总陨石数 | 玩家当前的吃力程度 |
| **移动轨迹平滑度** | 玩家移动是慌乱乱窜还是平稳有序 | 玩家是紧张还是放松 |
| **贴近躲避频率** | 玩家有多少次是擦着陨石边躲过的 | 判断难度是否到极限 |
| **存活时间趋势** | 最近几局的存活时间是上升还是下降 | 判断玩家是在进步还是退步 |

### 难度调整逻辑

```
玩家表现很好（躲避率高、移动平稳）→ 适度提高难度
  → 陨石下落速度 +5%
  → 陨石生成间隔 -8%
  → 或引入更大的陨石

玩家表现较差（躲避率低、移动慌乱）→ 适度降低难度
  → 陨石下落速度 -5%
  → 陨石生成间隔 +8%
  → 或减少大陨石出现概率

调整频率：每 5 秒评估一次，平滑调整，避免突变
```

### 为什么这算"AI"

虽然不是大模型，但核心思路是**让游戏像一个会观察、会调整的智能对手**——它不按固定剧本走，而是根据玩家的实时表现动态应对。这就是 AI "自适应"的思想在小游戏领域的落地。

## 七、全服排行榜设计

### 排行榜展示

- 展示**全服前 100 名**玩家的分数和用户名
- 显示玩家**自己的排名**（即使没进前 100 也能看到自己排第几）
- 前三名有特殊图标/颜色高亮

### 分数上传机制

```
玩家游戏结束 → 弹出"是否上传分数" → 已登录用户自动上传 →
  分数高于历史最高分 → 更新排行榜
  分数低于历史最高分 → 保留最高分
→ 显示当前排名和提升位数
```

### 冷启动策略

MVP 阶段没有大量用户时，加入若干**AI 虚拟玩家**的分数填充排行榜，让榜单看起来有竞争感。随着真实玩家增多，逐步替换掉 AI 玩家。

## 八、技术实现方案

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | Next.js 14 + React 18 + TypeScript |
| 游戏渲染 | Canvas 2D |
| 样式方案 | Tailwind CSS |
| 后端服务 | Supabase（数据库 + 认证） |
| 音频系统 | Web Audio API |
| 部署平台 | Vercel / Netlify |

### 游戏引擎架构

```
GameEngine（游戏主引擎）
├── PlayerSystem（飞船控制系统）
├── AsteroidManager（陨石管理系统）
├── AIDifficultySystem（AI 难度系统）← 核心
│   ├── 数据采集模块
│   ├── 难度评估模块
│   └── 平滑调整模块
├── CollisionDetector（碰撞检测系统）
├── ScoreSystem（计分系统）
├── AudioManager（音频管理）
├── UIManager（界面管理）
└── LeaderboardSystem（排行榜系统）
```

### 核心技术亮点

1. **像素级精准碰撞检测** — 通过离屏 Canvas 生成碰撞掩码，比矩形碰撞更精准
2. **60fps 流畅渲染** — requestAnimationFrame 驱动 + 对象池优化
3. **AI 难度算法** — 多维度数据分析 + 平滑调整策略
4. **全设备响应式适配** — 桌面键盘 + 移动触摸双方案
